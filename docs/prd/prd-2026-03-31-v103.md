# LumenPDF — 产品需求文档 (PRD)

**版本**: v1.0.3 · **日期**: 2026-03-31

---

## 本次迭代概述 (v1.0.3)

基于 v1.0.2 版本，本次迭代聚焦于以下优化：

1. 数据库 schema 变更规范
2. 划线颜色优化
3. 句子翻译保存逻辑优化

---

## 新增/变更功能需求

### O9 — 数据库 Schema 变更规范

**问题**：当前项目缺少数据库 schema 变更的规范，可能导致数据丢失或兼容性问题。

**需求**：

- 所有数据库 schema 变更必须向前兼容
- 使用 `sqlx` 框架和 `sqlx migrate` 工具进行数据库迁移
- 如必须打破兼容性，需提供迁移脚本确保数据不丢失
- 迁移脚本需记录版本号，支持回滚

**实现**：

- 在 CLAUDE.md 中添加数据库变更规范
- 使用 `sqlx migrate` 管理迁移脚本
- 迁移脚本存放在 `lumen-pdf-core/migrations/` 目录

### O10 — 划线颜色优化

**问题**：当前划线的红色不够醒目，与 macOS 原生 PDF 预览器的红色有差异。

**需求**：

- 划线颜色调整为与 macOS 原生 PDF 预览器一致的红色
- 红色应更加醒目和标准

**实现**：

- 将 `NSColor.systemRed.withAlphaComponent(0.7)` 改为更鲜艳的红色
- 使用 `NSColor(red: 1.0, green: 0.23, blue: 0.19, alpha: 0.85)` 作为划线颜色

### O11 — 句子翻译保存到笔记

**问题**：选中句子进行翻译时，保存按钮显示「保存到单词本」，与用户期望不符。句子翻译应该保存到笔记，而非单词本。

**需求**：

- 句子翻译模式下，按钮文本显示「保存到笔记」
- 保存时同时保存原文和译文
- 保存后自动添加划线标注

**实现**：

- 在 TranslationBubble 中根据 `isSentenceMode` 判断显示不同的按钮文本
- 句子模式保存时调用 `saveNote` API，将译文作为 `note` 字段保存
- 保存后触发划线标注通知
