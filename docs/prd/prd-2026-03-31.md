# LumenPDF — 产品需求文档 (PRD)

**版本**: v1.0.2 · **日期**: 2026-03-31

---

## 本次迭代概述 (v1.0.2)

基于 v1.0.0 版本，本次迭代聚焦于用户体验优化和功能增强，主要改进：

1. 翻译失败时的完整错误信息展示
2. 句子翻译模式（整句翻译而非单词分析）
3. 笔记功能与划线标注联动
4. 侧边栏交互优化
5. 标注按钮尺寸优化

---

## 新增/变更功能需求

### O1 — LLM 翻译失败时显示错误信息

**问题**：当 LLM 翻译失败但 MyMemory 兜底成功时，用户不知道 LLM 为什么失败，可能误以为是正常翻译结果。

**需求**：在翻译气泡底部显示 LLM 失败原因，即使兜底翻译成功。用户可以据此调整 API 配置。

**展示内容**：

- 当 LLM 失败时，气泡底部显示橙色警告条
- 内容示例：「LLM 调用失败：API Key 无效」或「LLM 调用失败：网络超时」

### O2 — LLM 和兜底翻译均失败时显示双重错误

**问题**：当两者都失败时，用户只看到一个模糊的错误提示。

**需求**：同时展示 LLM 失败原因和兜底失败原因，帮助用户排查问题。

**展示内容**：

- 气泡变为红色错误状态
- 显示「LLM 调用失败：xxx」
- 显示「兜底翻译失败：xxx」
- 提供「重试」按钮

### O3 — PDF 标注持久化

**问题**：自由标注（高亮/划线）仅在当前 Session 有效，App 重启后丢失。

**需求**：将所有标注持久化到 PDF 文件元数据中，重启后自动恢复，与 macOS Preview 行为一致。

**实现**：

- 使用 PDFAnnotation 的 `userName` 和 `contents` 属性存储标注信息
- 划线笔记标注使用笔记 ID 作为 `userName`，便于关联删除
- App 启动时从 PDF 元数据恢复所有标注

### O4 — 笔记功能（划线笔记联动）

**问题**：划线标注与笔记分离，无法关联管理。

**需求**：

- 选中文本后点击「划线笔记」，自动添加蓝色下划线并保存笔记
- 笔记内容：划线文本 + PDF 来源 + 页码 + 用户备注
- 删除笔记时，自动移除对应的划线标注

**数据结构**：

| 字段         | 说明                         |
| ------------ | ---------------------------- |
| `id`         | 笔记 UUID                    |
| `pdf_path`   | PDF 文件路径                 |
| `pdf_name`   | PDF 文件名                   |
| `page_index` | 页码（0-based）              |
| `content`    | 划线选中的文本               |
| `note`       | 用户添加的备注               |
| `bounds_str` | 选区矩形（NSStringFromRect） |

**导出功能**：

- 支持导出所有笔记为 Markdown 格式
- 支持导出单个 PDF 的笔记
- 导出内容包含：来源 PDF、页码、划线文本、用户备注

### O5 — 句子翻译模式

**问题**：选中短语或句子时，翻译气泡仍按单词格式展示，信息冗余且不自然。

**需求**：

- 当选区包含多个单词时，自动切换为「句子翻译」模式
- 气泡内容简化：只展示整句译文，不展示单词分析（音标、词性、语境解释等）
- 按钮区显示「句子翻译」徽标

**判断逻辑**：

- 选区文本包含空格 → 判定为句子/短语
- 调用专用 `translate_sentence` API，仅返回整句译文

### O6 — 操作按钮尺寸优化

**问题**：选区操作菜单按钮过小，点击命中率低。

**需求**：增大翻译、高亮、划线笔记按钮的点击区域，提升操作体验。

**实现**：

- 按钮 padding 增大
- 字体尺寸适当增大
- 保持视觉协调

### O7 — 侧边栏收起/展开

**问题**：NavigationSplitView 内置的侧边栏 Toggle 按钮无响应。

**需求**：

- 修复原生侧边栏 Toggle 按钮功能
- 用户可通过工具栏按钮或系统快捷键收起/展开目录栏

**实现**：

- 将 `columnVisibility` 从 `.constant(...)` 改为 `$columnVisibility` 双向绑定
- 保持 SwiftUI NavigationSplitView 原生交互

### O8 — 切换 Tab 时文件名保持可见

**问题**：切换到单词本或笔记 Tab 时，工具栏文件名消失，用户失去上下文。

**需求**：

- 只要选中有 PDF 文件，工具栏始终显示文件名
- 仅在 PDF 阅读 Tab 时显示页码指示器
- 单词本/笔记 Tab 时文件名独立显示，无页码

## 变更的功能需求（基于 v1.0.0 PRD）

### F2 — 选词操作菜单（变更）

| 操作         | 行为                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| **翻译**     | 开启翻译气泡，调用后端翻译；选中多词时自动切换句子翻译模式                  |
| **高亮**     | 在 PDF 添加黄色高亮标注（Toggle：再次点击则移除；与现有高亮部分重叠则合并） |
| **划线笔记** | 在 PDF 添加蓝色下划线标注，同时自动保存笔记；删除笔记时同步移除划线         |

### F5 — 原生 PDF 标注（变更）

- **高亮（Highlight）**：黄色，词汇高亮用 UUID `userName` 标记；自由标注用 `"__fh"` 标记。
- **划线（Underline）**：蓝色，划线笔记用笔记 ID `userName` 标记（便于关联删除）；自由划线用 `"__fu"` 标记。
- **持久化**：所有标注通过 PDF 元数据持久化，重启后自动恢复。

## 界面变更

### 工具栏

- 文件名始终显示（选中 PDF 时）
- 页码仅在 PDF 阅读 Tab 显示
- 侧边栏 Toggle 按钮响应正常

### 翻译气泡

- 句子翻译模式：简化布局，只显示整句译文
- 错误区：LLM 失败时显示橙色警告；双重失败时显示红色错误
- 按钮：点击区域增大

### 笔记 Tab

- 新增笔记列表视图
- 搜索、编辑、删除功能
- Markdown 导出功能
- 点击来源跳转到 PDF 对应页面
