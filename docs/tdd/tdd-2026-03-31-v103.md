### O9 — 数据库 Schema 变更规范

#### CLAUDE.md 更新内容

在 CLAUDE.md 中添加以下规范：

```markdown
## Database Schema Changes

所有数据库 schema 变更必须遵循以下规范：

1. **向前兼容**：新增字段必须有默认值或允许 NULL
2. **使用 sqlx migrate**：使用 `sqlx migrate add <name>` 创建迁移脚本
3. **迁移脚本位置**：`lumen-pdf-core/migrations/`
4. **破坏性变更**：如必须删除字段或修改类型，需提供数据迁移逻辑
5. **测试**：迁移脚本需在测试环境验证后再部署

### 迁移脚本示例

```sql
-- 新增字段（向前兼容）
ALTER TABLE notes ADD COLUMN translation TEXT DEFAULT '';

-- 数据迁移（破坏性变更）
CREATE TABLE notes_new (
    id TEXT PRIMARY KEY,
    pdf_path TEXT NOT NULL,
    pdf_name TEXT NOT NULL,
    page_index INTEGER NOT NULL,
    content TEXT NOT NULL,
    note TEXT NOT NULL,
    translation TEXT DEFAULT '',
    bounds_str TEXT NOT NULL,
    created_at INTEGER NOT NULL
);

INSERT INTO notes_new SELECT id, pdf_path, pdf_name, page_index, content, note, '', bounds_str, created_at FROM notes;
DROP TABLE notes;
ALTER TABLE notes_new RENAME TO notes;
```

### 命令

```bash
# 创建新迁移
cd lumen-pdf-core && sqlx migrate add <migration_name>

# 运行迁移
cd lumen-pdf-core && sqlx migrate run

# 回滚迁移
cd lumen-pdf-core && sqlx migrate revert
```

### O10 — 划线颜色优化

#### 当前实现

```swift
// PDFReaderView.swift:575-576
let color: NSColor = typeStr == "underline"
    ? NSColor.systemRed.withAlphaComponent(0.7)
    : NSColor.systemYellow.withAlphaComponent(0.5)
```

#### 优化方案

使用更鲜艳的红色，接近 macOS Preview 的划线颜色：

```swift
// PDFReaderView.swift
let color: NSColor = typeStr == "underline"
    ? NSColor(red: 1.0, green: 0.23, blue: 0.19, alpha: 0.85)
    : NSColor.systemYellow.withAlphaComponent(0.5)
```

#### 颜色对比

| 颜色 | RGB 值 | 效果 |
|------|--------|------|
| 当前红色 | systemRed + 0.7 alpha | 偏粉，不够醒目 |
| 优化后红色 | (1.0, 0.23, 0.19, 0.85) | 鲜艳的红色，接近 macOS Preview |

### O11 — 句子翻译保存到笔记

#### 当前实现

TranslationBubble 中的 footer 方法固定显示「保存到单词本」：

#### 优化方案

1. **TranslationBubble.swift 修改**

```swift
@ViewBuilder
private func footer(result: TranslationResult) -> some View {
    HStack {
        Spacer()
        if let entryId = savedEntryId {
            // 已保存状态...
        } else {
            Button {
                savedEntryId = onSave(result)
            } label: {
                // 根据 isSentenceMode 显示不同的按钮文本
                if request.isSentenceMode {
                    Label("保存到笔记", systemImage: "note.text")
                } else {
                    Label("保存到单词本", systemImage: "bookmark")
                }
            }
            .buttonStyle(.borderedProminent)
        }
    }
    .padding(.horizontal, 14)
    .padding(.vertical, 10)
}
```

2. **PDFReaderView.swift 修改**

```swift
// 修改 onSave 回调，根据 isSentenceMode 调用不同的保存方法
private func handleSaveTranslation(result: TranslationResult, request: TranslationBubbleRequest) -> String? {
    if request.isSentenceMode {
        return saveSentenceToNote(result: result, request: request)
    } else {
        return saveToDiary(result: result, request: request)
    }
}

private func saveSentenceToNote(result: TranslationResult, request: TranslationBubbleRequest) -> String? {
    BridgeService.shared.initializeIfNeeded()

    // 将译文作为 note 字段保存
    let translation = result.contextSentenceTranslation.isEmpty
        ? result.contextTranslation
        : result.contextSentenceTranslation

    guard let noteEntry = try? BridgeService.shared.saveNote(
        pdfPath: document.filePath,
        pdfName: document.fileName,
        pageIndex: UInt32(request.page),
        content: request.word,  // 原文
        note: translation,       // 译文
        boundsStr: request.boundsStr
    ) else {
        return nil
    }

    // 添加划线标注
    NotificationCenter.default.post(
        name: .addUnderlineNote,
        object: nil,
        userInfo: [
            "noteId": noteEntry.id,
            "pageIndex": request.page,
            "boundsStr": request.boundsStr,
            "filePath": document.filePath
        ]
    )

    appState.refreshNotes()
    appState.showToast("已保存到笔记")
    return noteEntry.id
}
```

```

saveSentenceToNote()
    ↓

    ↓
PDFView 添加划线标注
    ↓
刷新笔记列表
